## 一、逻辑视图列表页 · 页面定位（先统一认知）

**页面定位（改版后）**

> **逻辑视图列表 = 语义治理的“入口工作台”**
>
> 核心问题不是「我有哪些表」，而是：
>
> * 哪些表**还没开始语义理解**
> * 哪些表**卡在字段语义确认**
> * 哪些表**已经具备对象建模条件**

所以：
**这是一个“治理状态驱动 + 行动导向”的列表页，而不是纯资源列表。**

---

## 二、页面整体布局（Figma Frame 层级）

```
Frame: 逻辑视图列表页（Semantic View List）

├─ PageHeader
├─ FilterBar
├─ ListSummaryBar
├─ DataTable
│  └─ RowAction
└─ BulkActionBar（条件显示）
```

---

## 三、组件级信息结构（逐块拆解）

---

### 1️⃣ PageHeader（页面头部）

**Frame：PageHeader**

```
[ 页面标题 ]  逻辑视图
[ 副标题 ]    语义治理入口 · 从字段理解到业务对象建模
```

**右侧操作区**

```
[ 批量语义理解 ]（主按钮）
[ 帮助 / 指南 ]
```

> 🔑 改进点

* 把「语义治理入口」明确写出来，**用户一眼知道这是治理页，不是数据源页**
* “批量语义理解”是全局动作，而不是列表里的弱按钮

---

### 2️⃣ FilterBar（治理视角的筛选区）

**Frame：FilterBar**

```
[ 搜索框 ]  逻辑视图 / 业务名称

[ 治理状态 ]（下拉）
- 未开始语义理解
- 字段语义待确认
- 语义建模进行中
- 可进入对象建模
- 已完成

[ 风险状态 ]
- Gate 未通过
- 高风险
- 无风险

[ 数据源 ]（可选）
```

> 🔑 改进点

* **治理状态是第一维度，而不是“是否 Review”**
* Gate / 风险只作为辅因，不再主导页面认知

---

### 3️⃣ ListSummaryBar（治理总览条）

**Frame：ListSummaryBar（轻量 KPI）**

```
未开始 12 ｜ 待确认 8 ｜ 建模中 6 ｜ 可建模 4 ｜ 已完成 6
```

> 🔑 这是“治理盘感”，不是数据盘点
> 让用户知道：**当前卡在哪一步最多**

---

### 4️⃣ DataTable（核心表格）

#### 表头结构（改版后）

| 列          | 名称          | 说明      |
| ---------- | ----------- | ------- |
| □          | 选择框         | 批量操作    |
| 逻辑视图       | 物理名 + 业务名   | 保留你现有样式 |
| 数据源        | DB / Schema | 不变      |
| 行数         | 行数          | 不变      |
| **语义建模状态** | 核心          | ⭐ 新核心列  |
| **字段进度**   | 数值          | ⭐ 新     |
| Gate       | 质量/安全门槛     | 保留      |
| 风险         | 风险数         | 保留      |
| 更新时间       | -           | 保留      |
| 操作         | CTA         | ⭐ 强化    |

---

#### ⭐ 语义建模状态（关键列）

**状态枚举（统一语义语言）**

```
未开始语义理解
字段语义待确认
语义建模进行中
可进入对象建模
已完成
```

**视觉规范**

* 灰色：未开始
* 橙色：待确认
* 蓝色：进行中
* 绿色：可建模 / 已完成

> 🔑 这是你之前强调的
> **“把功能态 / 系统态 → 统一成语义建模语言”**

---

#### ⭐ 字段进度（辅助但关键）

展示方式：

```
5 / 12 字段已确认
```

或 Hover：

```
已确认：5
待确认：7
```

> 🔑 用户不用点进去，就知道“卡在字段层面”

---

### 5️⃣ RowAction（行级 CTA 设计）

**操作列不再是「开始理解 / 继续理解」这种模糊词**

而是 **状态 → 唯一下一步动作**

| 当前状态    | CTA 文案   |
| ------- | -------- |
| 未开始语义理解 | 开始字段语义理解 |
| 字段语义待确认 | 继续字段语义确认 |
| 语义建模进行中 | 查看语义进展   |
| 可进入对象建模 | 进入对象建模   |
| 已完成     | 查看语义结果   |

> 🔑 这是**列表页最重要的改进**
>
> * 用户不需要“想下一步”
> * 系统直接告诉你「现在该干嘛」

---

### 6️⃣ BulkActionBar（批量操作）

**仅在多选时出现**

```
已选 3 个逻辑视图
[ 批量开始字段语义理解 ]
```

> ⚠️ 仅支持**同状态批量**
>
> * 防止语义流程被破坏

---

## 四、与字段语义理解页的衔接关系（关键）

**点击 CTA 后的行为**

```
逻辑视图列表
  → 直接进入
     字段语义理解 · 表级页面
     （默认 Tab：字段结构）
```

> ❗ 注意
>
> * **不跳新菜单**
> * **不切上下文**
> * 用户始终感觉：
>   “我就是在治理这个表”

---

## 五、为什么「逻辑视图列表页必须保留，但要升级」

结论非常明确：

> **必须保留，且必须升级为“治理调度页”**

### 如果删除 / 弱化，会发生什么？

* 用户只能在字段页“埋头干活”
* 无法回答：

  * 我还有多少表没治理？
  * 哪些卡在字段确认？
  * 哪些已经能建对象？

### 升级后你得到什么？

* 一个**语义治理节奏控制台**
* 为后续接入：

  * 业务对象建模
  * 质量 Gate
  * 安全等级
    打下统一入口
